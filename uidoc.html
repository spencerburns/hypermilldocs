Module om.cam.ui¶

How to include¶

To include ‘om.cam.ui’ your Python code, place these lines in your code.

import om.cam.core
import om.cam.ui

om.cam.ui always needs the core module om.cam.core too!

Examples¶

Here you find some simple examples showing how to generate a basic UI using the om.cam.ui module.

  1. Single Parameter : Demonstrates how to query the user for a single parameter, e.g. a filename

  2. Parameter List : Shows how to query the user for a whole list of different values, each with a different type

  3. Parameter Group : Group parameters under a common name and create a UI for the group

  4. Parameter Group List : Create lists of parameter groups and create a UI for the list

  5. Enums : Shows how to use pre-defined enums in the UI or how to define enums in Python itself

Single Parameter¶
import om.cam.core
import om.cam.ui
import sys

if __name__ == '__main__':
    
    # To be able to display any dialog, Hypermill has to be running.
    if not om.cam.core.IsHyperMillRunning():
        sys.exit(1)
   
    # For convencience some shortcuts for namespaces: 
    ns = om.cam.ui
    pt = ns.PyParameterType
    pd = ns.PyParameterDescription

    # A single parameter of type "String" with the name "File Name" is defined
    fileNameParameter = pd(pt.String, "File Name")
    # The current value for each parameter can be set using the "SetValue" function.
    fileNameParameter.SetValue("This could be your file name!")

    # A dialog is created to query for the single parameter.
    # The title of the dialog will be "Open File"
    dialog = ns.PyCreateParameterDialog("Open File", fileNameParameter)

    # The dialog is now shown as modal dialog, the script will not continue until the user either accepts or cancels the dialog.
    # Show() will return PyDialogResult::Accepted, PyDialogResult::Rejected or PyDialogResult::Error.
    dialogResult = dialog.Show()
    if(dialogResult == ns.PyDialogResult.Accepted):
        # If the user accepted (pressed "OK") and no error occured, the parameter description for the parameter given by the user
        # can be queried using the name of the parameter:
        modifiedParameter = dialog.NameDictionary()["File Name"]
        # Now you can access the actual value provided by the user:
        fileName = modifiedParameter.Value()

Parameter Group List¶
import om.cam.core
import om.cam.ui
import sys

if __name__ == '__main__':
    # This example shows how the user can create several parameter groups and combine them into a parameter group list.
    # The group list is then used to create a dialog to query the values for the parameters.
    
    # To be able to display any dialog, Hypermill has to be running.
    if not om.cam.core.IsHyperMillRunning():
        sys.exit(1)
   
    # For convencience some shortcuts for namespaces: 
    ns = om.cam.ui
    pt = ns.PyParameterType
    pd = ns.PyParameterDescription
    pdl = ns.PyParameterDescriptionList
    pg = ns.PyParameterGroup
    pgl = ns.PyParameterGroupList

    # In this example two 3D coordinates should be queried: a start point and an end point
    # First defining the coordinates for start and end point
    pStartX = pd(pt.Float64, "X")
    pStartY = pd(pt.Float64, "Y")
    pStartZ = pd(pt.Float64, "Z")
    pEndX   = pd(pt.Float64, "X")
    pEndY   = pd(pt.Float64, "Y")
    pEndZ   = pd(pt.Float64, "Z")

    # Now creating groups for the start and and point
    pStart = pg("Start", [pStartX, pStartY, pStartZ])
    pEnd   = pg("End", [pEndX, pEndY, pEndZ])

    # Now creating the group list to be used
    pGroupList = pgl([pStart, pEnd])

    # A dialog is created to query the parameters in both groups.
    # The title of the dialog will be "Set Translation"
    dialog = ns.PyCreateParameterDialog("Set Translation", pGroupList)

    # The dialog is now shown as modal dialog, the script will not continue until the user either accepts or cancels the dialog.
    # Show() will return PyDialogResult::Accepted, PyDialogResult::Rejected or PyDialogResult::Error.
    dialogResult = dialog.Show()

    if(dialogResult == ns.PyDialogResult.Accepted):
        # If the user accepted (pressed "OK") and no error occured, the new values for origin can be queried.
        # We can access each parameter description in a parameter group using the "Descriptions" function:
        for description in pStart.Descriptions():
            print(description.Name() + " : " + str(description.Value()))
        print(pEnd.Name())
        for description in pEnd.Descriptions():
            print(description.Name() + " : " + str(description.Value()))
Using Enums¶
import om.cam.core
import om.cam.ui
import sys

# If you want to define your own enums in Python, use the "enum" module provided by the Python Standard Library
import enum


if __name__ == '__main__':
    
    # To be able to display any dialog, Hypermill has to be running.
    if not om.cam.core.IsHyperMillRunning():
        sys.exit(1)
   
    # For convencience some shortcuts for namespaces: 
    ns = om.cam.ui
    pt = ns.PyParameterType
    pd = ns.PyParameterDescription
    pdl = ns.PyParameterDescriptionList

    # Custom Python Enums
    # Here we define our own enum (simply some different kinds of animals) purely in Python, no corresponding enum exists in C++.
    # For more details how to define enums in Python, please consult the documentation for the "enum" module in the Python Standard Library.
    Animal = enum.Enum('Animal', [('CAT', 3), ('DOG', 5), ('DUCK', 7), ('ELEPHANT', 13), ('PIG', 17)])
    currentAnimal = Animal.DUCK
    # To allow the user, to select a kind of animal, a parameter description of type "Enum" has to be created.
    parameterOne = pd(pt.Enum, "Some Animals", currentAnimal)

    # Predefined Enums
    # If you want to use an enum that is already exposed in the Python API,  simply create a variable of this type,
    # and then use it to create a PyParameterDescription with type "enum".
    buttonValue = om.cam.core.PyAlertButton.Yes
    parameterTwo = pd(pt.Enum, "Button Value", buttonValue)
    # To allow the user to select both parameters, put them into a PyParameterDescriptionList
    parameterList = pdl([parameterOne, parameterTwo])

    # A dialog is created to query for the parameters create for the enums.
    # The title of the dialog will be "Using Enums"
    dialog = ns.PyCreateParameterDialog("Using Enums", parameterList)


    # The dialog is now shown as modal dialog, the script will not continue until the user either accepts or cancels the dialog.
    # Show() will return PyDialogResult::Accepted, PyDialogResult::Rejected or PyDialogResult::Error.
    dialogResult = dialog.Show()
    if(dialogResult == ns.PyDialogResult.Accepted):
        print("Selected animal is: " + str(parameterOne.Value()))
        # print("Selected button value is: " + str(parameterTwo.Value()))

class om.cam.ui.PyParameterDescription(PyObject)¶

A PyParameterDescription contains the information necessary to query or display a parameter in a UI, like its name, type, default value, tooltip and so on. To change a single parameter using a PyParameterDescription use a PyParameterDialog or the PyCreateParameterDialog function with a PyParameterDescription as argument.

  • theName (str) – The name of the parameter as shown to the user.

Initializes an PyParameterDescription with a type, a name and a default value.

class om.cam.ui.PyParameterGroup¶

A PyParameterGroup contains a list of parameters that are used together under a common name, e.g. the “Red”, “Green” and “Blue” components of a color. To change the parameters in a PyParameterGroup, use a PyParameterGroupDialog or the PyCreateParameterDialog function with a PyParameterGroup as argument.
